Estrutura Básica dos Diretórios

Como solicitado, criei esta estrutura básica para que possamos dar prosseguimento ao projeto.

Inseri vários metadados no index, mas comentei alguns por serem opções extras. Caso sejam desnecessários, podem excluí-los.

Explicação das pastas

  • O assets serve para inserirmos as imagens, ícones, arquivos, vídeos e qualquer outro tipo de arquivo necessário que sirva de material para o site ou para as páginas. Se necessário, criem novas páginas para agrupar os arquivos de modo que fiquem sempre organizados.
  • O css serve para inserir códigos CSS. Inicialmente inseri um arquivo de normalize para normalizar todas as páginas do projeto (se quiser saber mais, clique aqui) e criei o arquivo app.css para inserir o CSS padrão para todas as páginas. Conforme mais páginas forem sendo criadas, criem um novo arquivo CSS com o nome da nova página para servir de estilização exclusiva para essa nova página.
  • O js serve para inserir código JavaScript. Criei o arquivo app.js para inserir as funções padrão para todas as páginas. Conforme mais páginas forem sendo criadas, criem um novo arquivo JavaScript com o nome da nova página para inserir funções exclusivas para essa nova página.
  • O pages serve para inserir as páginas HTML referentes aos seminários e minicursos do evento.
  • Esta é a página inicial. O código dela deve ser incrementado para servir de página de apresentação do site.
